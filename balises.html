Les titres

La taille est de plus en plus petite plus le niveau augmente

  • h1 : Titre principal
  • h2 : Titre de niveau 2
  • h3 : Titre de niveau 3
  • h4 : Titre de niveau 4
  • h5 : Titre de niveau 5
  • h6 : Titre de niveau 6
  • title : ajoute une définition sur un élément au passage de la souris, allez, passez la souris sur le titre de ce paragraphe pour voir =)

Les éléments qui composent une page

  • head : fournit des informations relatives à la page et comprend généralement un élément "title"
    • [!DOCTYPE html] : placé dans le head de la page, indique le type de document au navigateur
  • title : titre affiché sur la barre du navigateur, de l'onglet
  • body : contenu qui doit être affiché dans la fenetre du navigateur, le "corps" de la page
  • p : défini un paragraphe, un espace sera défini entre chaque paragraphe

On peut définir des attributs au paragraphe, en les ajoutants à la balise. par exemple : [p lang="fr"]indique que le text du paragraphe est en français.

Mise en forme du texte

  • strong : mets le texte en gras, indique que le texte est d'une grande importance
  • b : caractères gras aussi, mais n'as pas la meme valeure pour un lecteur d'ecran
  • i : caractères en italique
  • sup : caractères devant etre placés en exposants
  • sub : caractères devant etre placés en inverse de exposant, mais j'ai plus le nom
  • ins : souligne le texte
  • del : barre le texte
  • s : défini un contenu qui n'est plus exact mais qui ne doit pas être supprimé
  • font : définissais un paramêtre d'écriture, genre la couleure, avant CSS. N'est plus utilisé mais fonctionne toujours

Mise en page générale

  • br\ : ajoute un retour à la ligne sans lignes vides
  • hr\ : ajoute une ligne horizontale de séparation
  • em : indique une accentuation qui change subtilement le sens de la phrase. par défaut affiché en italique
  • blockquote :
    citations qui occupent tout un paragraphe, possibilité d'ajouter la source dans le bloque. [blockquote cite="source"]
  • q : s'emploie avec les citations courtes, les navigateurs sont sensés placer des guillemets autour du contenu.
  • abbr : permet de définir une abbreviation avec vue de la définition au passage de la souris. -> Le Pr Stephen truc = [abbr title="Professeur"]Pr[/abbr]
  • cite : en cas de références à un autre document, mets le text en italique [cite]blablabla[/cite]
  • dfn : sert à indiquer la définition d'un nouveau terme. Une définition est un truc sympa
  • style : peut s'insérer dans les balises pour définir un style à un élément, comme le fond de celui-ci. ( style="")
    • background-color: : défini la couleur de l'arrière fond de l'élément
    • color: : défini la couleur du texte de tout l'élement (h1, body, p, li,..)
  • address : sert à indiquer les coordonnées de l'auteur de la page comme son mail ou son adresse, affich; en italique

    • [address][p][a href="mailto:xxx"]xxx[/a][/p] [p]742 evergreen terasse[/p] [/address]

  • !-- : permet de mettre des annotations sur le code qui ne seront pas prises en compte par le navigateur

Les listes

  • ol : défini une liste ordonnée et numerotée
  • ul : défnini une liste non ordonnée et donc non numerotée, comme celle ci
  • li : défini les éléments de la liste
  • dl : défini une liste de définitions
    • dt : défini le terme à .... bah définir
    • dd : défini la définition

    exemple: [dl] [dt]mot[/dt] [dd]défintion du mot[/dd] [/dl]

    Mot
    Définition du mot

on peut avoir des sous-listes en remettant un [ul](ou autre). Voir l'exemple sur cette page

Les liens

url absolue = adresse complète menant à un site précis
url relative = le nom d'une page du même site, le système de classement des liens de dossier est pareil que sur windows :

  1. le document est dans un sous dossier du dossier ou l'on se trouve : "dossier1/dossier2/dossier3/document.html
  2. le document est dans un dossier "parent": "../../../document.html

  • a : création d'un lien
  • href="" : défini la cible du lien
  • target : permet de définir la fenêtre de destination (_blank = nouvelle fenêtre)
  • mailto : envoye un mail à l'adresse définie. [a href="mailto:adresse@truc.com"]Message affiché[/a]

Les images

Des exemples peuvent êtres trouvés sur cette page

  • img : ajoute une image
  • src : précise l'emplacment de l'image (un site ou un dossier du site)
  • alt : description textuelle de l'image, utile pour les recherches google
  • title : ajoute un texte visible en passant la souris sur l'image
  • height : indique la hauteur de l'image en pixels
  • width : indique la largeur de l'image en pixels
  • figure : permet de grouper un commentaire avec une ou plusieurs photos
  • ficaption : défini le commentaire associé aux images
  • align : utilisée dans les anciennes versions de HTML, aujourd'hui elle est remplacée par du CSS

[img src="source de l'image" alt="hey google trouve moi" title="le texte au passage de la souris height="100" width="100"/]

La position de l'image par rapport au texte peut aussi être définie, comme démontré ici

Les tableaux

Des exemples peuvent êtres trouvés sur cette page

  • table : création du tableau
  • tr : défini une ligne du tableaux (Table Row)
  • td : défini une cellule du tableau (Table Data)
  • th : défini une en-tête du tableau (Tables Head)
    • scope="col" : indique que l'en-tête concerne une colonne
    • scope="row" : indique que l'en-tête concerne une ligne
  • colspan="" : défini le nombre de colonnes occupées par une cellule
  • rowspan="" : défini le nombre de lignes occupées par une cellule
  • thead : en-tête d'un tableau (long)
  • tbody : corps d'un tableau (long)
  • tfoot : pied d'un tableau (long)

Les formulaires

Des exemples peuvent êtres trouvés sur cette page

  • form : élément définissant un formulaire
  • action : tout élément [form] à besoin d'un attribut action, précise l'URL de la page qui recevra les informations du formulaire
  • method : il existe deux méthodes d'envoi de formulaires :
    • get : les valeures saisies sont ajoutées à la fin de l'url [action], utile pour une recherche google par exemple, récupère des données sans en ajouter
    • post : les valeures sont envoyées dans une en-tête http, pour formulaires longs, envoyer un fichier, comprend des données sensibles, ajoute des infos dans une base de données
  • id : sert à identifier le formulaire parmis les autres éléments de la page
  • size : servais à définir la valeure de champ de texte, remplacé par CSS
  • maxlength : limite le nombre de caractères acceptés dans un champ
  • name : chaque contrôle d'un formulaire exige un attribut "name" pour les différencier
  • placeholder : défini un texte affiché dans la zone de saisie en attendant une entrée
  • requierd="required" : indique au navigateur que le champ doit obligatoirement etre rempli pour envoyer le formulaire
  • textarea : permet de créer une zone de texte
  • select : liste déroulante
    • name : nom envoyé au serveur avec les réponses
    • option : options disponibles
      • value="" : valeure réponse qui sera envoyée au serveur
      • selected="selected" : valeure sélectionnée par défaut au chargement de la page
    • size : précise le nombre d'options à afficher dans le menu déroulant
    • multiple="multiple" : permet de choisir plusieurs réponses
  • label : etiquette de contrôle
    • for : précise le contrôle auquel s'applique l'étiquette
  • fieldset : regroupe des éléments d'un formulaire
    • legend : placé directement après, permet de décrire l'objet du groupe
  • input : créer différents contrôles de formulaires, la valeure de son attribut "type"détermine le type du champ qui sera ajouté
    • type="text" : crée un champ de saisie d'une ligne de text
    • type="password" : crée un champ de saisie mais cache les caractères qui sont introduits
    • type="radio" : permet de sélectionner une option parmis plusieurs
      • name : la valeure doit être identique pour l'ensemble des boutons "radio" d'une même question
      • value : la valeure qui sera envoyée au serveur quand l'option est cochée
      • checked="checked" : indique quel bouton est coché de base au chargement de la page, un par question
    • type="checkbox" : permet de sélectionner une ou plusieurs options
      • name : la valeure doit être identique pour l'ensemble des choix d'une même question
      • value : la valeure qui sera envoyée au serveur quand l'option est cochée
      • checked="checked" : indique quelles cases seront cochées de base au chargement de la page, plusieurs possibles par question
    • type="file" : créé un champ avec un bouton "parcourir"permettant de joindre un fichier
    • type="submit" : créé un bouton d'envoi pour soumettre le formulaire au serveur
      • value : détermine le texte qui s'affiche dans le bouton
    • type="image" : utiliser une image comme bouton d'envoi
    • type="date" : permet de choisir une date dans un calendrier
    • type="email" : optimisé pour la saisie d'e-mail, le navigateur contrôle le format pour valider le formulaire
    • type="url" : pareille que "email" mais pour les url
    • type="search" : défini un champ de recherche
    • type="hidden" : permet d'ajouter des informations qui seront envoyées en même temps que le formulaire mais qui n'apparaissent pas sur la page

Videos et Audio

[video src="" poster="" width="" height="" preload controls loop] [p]description de la video[/p][/video]

  • video : reconnait plusieurs attributs pour le contrôle de la lecture de la video
  • src : précise le chemin de la vidéo
  • poster : permet d'afficher une mignature affichée pendant le chargement de la video
  • width, height : defini la taille de la video

les attributs peuvent ne pas avoir de valeures, comme ici. Ces attributs sont comparables a des interupteurs marche/arret. Si l'attribut est présent, l'option correspondante est activée :

  • preload : indique au navigateur ce qu'il doit faire lorsque la page se charge. Voici les trois valeurs reconnues :
    • none : le chargement de la video doit débuter suite a l'appui sur le bouton de lecture
    • auto : la video est telechargee des le chargement de la page
    • metadata : seules les informations generales sur la video sont recuperees
  • controls : indique que le navigateur doit fournir ces propres controles pour la lecture
  • autoplay : stipule que la lecture doit demarrer automatiquement
  • loop : precise que la lecture doit se faire en boucle

[audio src="" controls autoplay] [p]message a afficher si le lecteur n'est pas pris en charge[/p][/audio]

les attributs sont simmilaires aux videos (src, controls, autoplay, preload, loop)

Balisages autres

  • id="" : permet d'identifier un élément de niveau bloc de la page. Utile pour faire un lien vers cet élément ou pour la mise en forme CSS
  • class="" : un peu pareil que "id" mais peut contenir plusieures classe à l'intérieur de "" ("class1 class2 class3")
  • niveau bloc : éléments qui seront toujours affichés sur une nouvelle ligne dans le navigateur (ex: h1, p, ul, li,...)
  • niveau texte : toujours affichés sur la même ligne que les éléments qui l'entourent. (ex : a, b, em, img,...)
  • div : permet de regrouper un jeu d'élements dans une boite de niveau bloc. En ajoutant un attribut "id' permet de créer des regles CSS pour tout un bloc.
  • span : equivalent en-ligne de l'element div. Permet de contenir une portion de texte.
  • iframe : cadre incorporé (Voir exemple ici
    • src : précise l'url de la page a afficher dans le cadre
    • height : precise la hauteur du cadre en pixels
    • width : precise la largeur du cadre en pixels
    • seamless : peut etre incorporer pour indiquer que les barres de defilement ne sont pas souhaitees
  • meta name="" : reside dans l'element head et comprend des informations relatives a la page web. N'as pas besoin de balaise de fermeture. Informations fournies par ces attributs :
    • Des exemples sont visibles dans le code source de cette page
    • Syntaxe : [meta name="description" content="bah la description lol"/]
    • description : contient une description de la page dont les moteurs de recherches se servent pour referencer la page
    • keywords : contient une liste de termes (tags) separes par des virgules. Permet de trouver la page
    • robots : indique si les moteurs de recherche doivent inclure la page.
      • noindex : stipule qu'elle ne doit pas être presente dans les resultats
      • nofollow : indiquer que les moteurs de recherche peuvent l'ajouter, mais sans les pages liees qu'elle comprend
  • meta http-equiv="" : couple d'attributs et contents :
    • author : indique le créateur de la page
    • pragma : evite que le navigateur place la page dans son cache, autrement dis, il ne la conserve pas localement
    • expires : permet d'indiquer a quel moment la page doit expirer et ne plus etre conservee dans le cache
