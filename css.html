Explications

Pour comprendre le fonctionnement de CSS il faut imaginer qu'une boite invisible entoure chaque element HTML. Il existe deux types d'elements distinct :

  1. niveau bloc : elements qui commencent sur une nouvelle ligne (h1 à h6, p, body, div,...)
  2. niveau text : elements qui restent de le flux du texte et nhe commencent pas sur une nouvelle ligne (b, i, ins, img, em, span,...)

Grace a CSS, nous pouvons ajouter une bordure autour de chaque boite, préciser sa largeur et sa hauteur, et definir sa couleur d'arriere plan. Nous pouvons aussi modifier l'aspect du texte dans la boite comme sa couleur, sa taille et sa police.

CSS se fonde sur l'association de regles aux elements HTML. ces regles regissent l'affichage des elements indiques et comprennent deux parties :

  1. un sélecteur : cible l'element auquel la regle s'applique. La meme regle peut concerner plusieurs elements si ceux-ci sont separés par des virgules.
  2. une declaration : précise le style des elemements designes par le selecteur. Chaque declaration est constituée d'une propriete et sa valeure. Les differentes declarations sont séparees par des points virgules.

Styles CSS externes : Utilise une page annexe .css liee pour definir les styles. Une page HTML peut employer plusieurs feuille de style CSS, il suffit de définir un element [link] pour chaque fichier CSS

  • link : indique au navigateur l'emplacement du fichier CSS qui definit les styles de la page. Sans balise de fermeture. Place dans l'element [head] de la page
  • type : indique le type du document lie. Sa valeure doit etre "text/css"
  • rel : definit la relation entre la page HTML et le fichier lié. Sa valeure doit etre "stylesheet" dans le cas d'un lien vers un fichier CSS.

Styles CSS internes : les regles CSS peuvent egalement etre declarées dans la page HTML en les insérant dans un element [style], generalement place dans l'element. L'element [style] doit utiliser l'attribut type pour indiquer que les styles sont definis en CSS. Sa valeure doit etre : [style type="text/css"] parametres [/style]

Les selecteurs CSS

Il existe differents types de selecteurs CSS pour que vous puissiez appliquer des règles à des éléments précis d'un document HTML

Selecteur Description Exemple
Selecteur universel S'applique a tous les élements du document * {} Cible tous les éléments de la page
Selecteur de type Correspond aux noms des éléments h1, h2, h3 {} Ciles les éléments [h1], [h2] et [h3]
Selecteurs de classe Correspond aux elements dont la valeure de l'attribut class est celle indiquee apres le point. .note {} Cible tous les elements dont l'attribut class a la valeure note
p.note {} Cible uniquement les elements [p] dont l'attribut class a la valeure note
Selecteur d'identifiant Correspond a l'element dont la valeure de l'attribut id est celle indiquée apres le symbole diese #introduction {} Cible l'element dont l'attribut id a la valeure introduction
Selecteur d'enfant Correspond aux elements qui sont les enfants direct d'un autre li>a {} Cible tous les elements [a] qui sont des enfants d'un element [li] (mais sans les autres elements [a] de la page
Selecteur de descendant Correspond aux elements qui sont des descendants de celui indique (pas seulement ces enfants directs) p a {} Cible tous les elements [a] qui se trouvent dans un eleement [p] meme ceux a plusieurs niveaux d'imbrication
Selecteur de freres adjacent Correspond a l'element enfant qui suit immediatement un autre enfant de meme parent h1+p {} cible le premier element [p] qui vient apres un element [h1] (sans les elements [p] suivants)
Selcteur de frere general Correspond aux elements qui sont les freres d'un autre, meme s'ils ne le suivent pas directement h1~p {} Si deux elements [p] sont les freres d'un element [h1], la regle s'applique aux deux

Cascade des règles CSS

  • Dernière règles : Si deux selecteurs sont identiques, le dernier est prioritaire
  • Spécificité : Si un selecteur est plus specifique que les autres, les regles les plus precises sont prioritaires
    • h1 est plus precis que *
    • p b est plus precis que p
    • p#intro est plus precis que p
  • Important : en ajoutant !important apres la valeure d'une propriete, on indique qu'elle doit etre consideree plus importante que les autres regles qui s'appliquent au meme element

Héritage

Si vous definissez les proprietes font-family ou color sur un element [body], elles s'appliquent a la plupart des elements enfants. En effet, la valeur de la propriete font-family est herite par les elements enfants. Cela vous evite d'appliquer ces proprietes a un grand nombre d'elements et conduit a des feuilles de style plus simple

Ce principe ne concerne pas certaines autres proprietes, come background-color ou border, qui ne sont pas heritees par les elements enfants. Si c'etait le cas, la page aurait probablement un aspect quelque peu neglige.

Les couleurs

  • color : precise la couleur du texte d'un element
  • background-color : CSS traite chaque element HTML comme s'il apparaissait a l'interieur d'une boite. Permet de modifier la couleur d'arriere plan du fond de la boite.
  • opacity : permet de preciser la transparence d'un element et de ses enfants
  • rgba : permet d'affecter une couleur, a la maniere d'une valeur RVB, mais avec une quatrieme composante pour la transparence. (0,0,0,0.5). Cette valeure alpha est un nombre entre 0 et 1. La valeure de RGBA affecte seulement l'element auquel elle est appliquee.
  • hsl : autre expression des couleurs. La valeure commene par les lettres hsl, suivies, entre paranthese, de la valeure de chaque composante :
    • Teinte : (hue) s'exprime sous forme d'un angle entre 0 et 360 degres.
    • Saturation : elle est donnee sous forme de pourcentage.
    • Luminosite : la luminosite est indiquee sous forme de pourcentage, 0% correspondant au blanc, 50% a la couleur normale, 100% au noir
  • hsla : permet de definir un e couleur comme precedement, a partir d'une teinte, d'une saturation et d'une luminosite, mais ajoute une quatrieme valeur pour la transparence (a l'instar de rgba)
    • Alpha : cette valeur est un nombre entre 0 et 1.0. Par exemple, 0.5 correspond a une transparence de 50% et 0.75, a une opacite de 75%

Texte

  • font-family : permet d'indiquer la police dans laquelle doit etre affiche le texte des elements selectionnes par la regle CSS. La valeure de cette propriete est le nom de la police choisie. On peut fournir une liste de police separees par des virgules afin que, si l'internaute ne dispose pas de votre premier choix, le navigateur puisse essayer l'une des autres polices de la liste. Lorsque le nom d'une police comprend plusieurs mots, il doit etre place entre guillements.
  • font-size : permet de preciser la taille de la police. Il existe plusieurs manieres de proceder :
    • pixels : la taille est souvent indiquee en pixels car les designers web ont ainsi un controle tes precis sur l'espace occupé par le texte. Le nombre de pixels est suivi des lettres px.
    • pourcentage : Dans les navigteurs, le texte a une taille par defaut fixee a 16px. Par consquent, une taille de 75% equivaut a 12px,.... Si une regle donne au texte de l'element [body] une taille de 75% (12px) et si une autre regle precise que le contenu d'un element dans l'element [body] doit avoir une taille de 75% ce contenu sera affiche en 9px (75 % de 12px)
    • em : un em est quivalent a la largeur de la lettre m. (16px = 1em)
  • @font-face : permet d'utiliser une police meme si elle n'est pas sur l'ordinateur de l'internaute. Pour cela, indiquer le chemin vers un exemplaire de cette police. La polie souhaitee est ajoutee a la feuille de style via une regle @font-face (voir code source de cette page)
    • font-family : precise le nom de la police. Il peut ensuite etre emplooye comme valeure de la propriete font-family dans les autres regles.
    • src : indique le chemin de la police.
  • font-weight : permet de mettre du texte en gras. Deux valeurs sont possibles : normal, bold
  • font-style : texte en italique. Troisw valeurs possibles : normal, italic, oblique (si italic existe pas pour la police, force le penchage)
  • text-transform : permet de modifier la casse du texte
    • uppercase : le texte est affiche en majuscule
    • lowercase : le texte est affiche en minuscule
    • capitalize : La premiere lettre de chaque mot est en majuscule
  • text-decoration : precise la decoration du texte
    • none : toute application deja appliquee au texte est retiree
    • underline : une ligne apparait sous le texte
    • overline : une ligne apparait au-dessus du texte
    • line-through : les mots sont barre d'un trait
    • blink : le texte clignote, mais cet effet est generalement deconseille car genant
  • line-height : interligne. (en em)
  • letter-spacing : espace entre les lettres. (en em)
  • word-spacing : espace entre les mots. (en em)
